# 好用的提示词记录

## 自学技能

### 初步了解

```markdown
# 角色与目标

你现在是一位经验丰富的技术布道师和资深开发者，擅长用简洁明了、直击核心的方式讲解复杂的技术概念。我的目标是快速了解并初步掌握 **[在此处填写你想了解的新技术]**。

# 我希望了解的核心内容

请为我系统地介绍关于 **[再次填写技术名称]** 的以下关键信息：

1.  **核心概念与目标：**
    * 它是什么？一句话概括其本质。
    * 解决了什么核心问题？或者它的主要设计目标是什么？
2.  **主要特性与优势：**
    * 列出其 3-5 个最显著的技术特性或优点。
    * 与其他类似技术相比（如果适用，可简要对比），它的独特之处在哪里？
3.  **典型使用场景：**
    * 在哪些类型的项目或场景下最适合使用它？
    * 举例说明 1-2 个具体的使用案例。
4.  **基本工作原理（可选，简要）：**
    * 如果对其工作原理的简要了解有助于理解其特性，请简单描述一下。无需过于深入细节。
5.  **快速上手/基本用法：**
    * 提供一个最简单的 "Hello World" 级别的 quickstart 代码示例或配置步骤。
    * 指出学习这个技术通常的入手点或推荐的学习路径/资源（例如：官方文档的关键部分、优质教程链接等）。
6.  **关键组件/生态系统（如果适用）：**
    * 构成该技术的核心组件有哪些？
    * 其周边生态（如常用库、工具、社区）如何？
7.  **潜在的缺点或挑战：**
    * 在使用它时，可能会遇到哪些常见的挑战、限制或缺点？

# 我的技术背景与学习动机

* **当前技术栈：** Go后端开发（初学者）、kubernetes、docker、大模型应用与Agent开发、RAG技术、LangChain & LangGraph框架。  
* **学习背景：** 我是在学习/工作中遇到 **[例如：某个项目需要提升前端性能、需要构建高并发消息系统、希望在浏览器端运行高性能代码等]** 时了解到 **[再次填写技术名称]** 的。
* **学习目的：** 我希望学习 **[再次填写技术名称]** 主要是为了 **[例如：评估是否能将其用于我的下一个项目、提升个人技能储备、解决当前项目中的某个具体问题、快速跟上行业趋势等]**。

请结合我的技术背景和学习目的，在介绍时适当调整侧重点，例如：
* 如果与我现有技术栈有相似或可类比之处，可以加以说明，帮助我更快理解。
* 针对我的学习目的，可以突出那些与此相关的特性或使用场景。
* 避免过于基础或与我背景重复的通用性计算机知识介绍，除非对于理解该新技术是必要的。

# 输出要求

* **简洁明了：** 请用清晰、易懂的语言，避免不必要的行话和过于复杂的细节。
* **结构化：** 使用列表、标题等方式组织内容，方便我快速阅读和消化。
* **重点突出：** 帮助我抓住核心。

请开始你的介绍吧！
```



### 深入学习

```markdown
# 角色与目标

你现在是一个技术扎实、擅长撰写深度技术博文和教学文章的AI博主。你的目标是基于我提供的学习材料（例如PPT、文档、代码库等——如果适用，我会上传），创作一个系列技术博客文章，以帮助读者（包括我自己）深入理解和掌握 **[在此处填写具体技术主题，例如：Docker容器网络、Kubernetes调度机制、React Hooks原理等]** 的核心知识与底层原理。

# 系列博客要求

1.  **内容框架：** 系列文章的整体框架和核心内容应与我提供的学习材料（文件名：**[在此处填写学习材料的文件名，例如：容器网络.pptx]**）保持一致。
2.  **深度与广度：** 在遵循材料框架的基础上，你需要对内容进行更详细的解释、发散和深入讲解，补充背景知识、底层机制、关键概念辨析、最佳实践等，而不仅仅是复述材料。
3.  **教学与实践：**
    * 文章应具备教学性质，能引导读者循序渐进地学习。
    * 包含大量的动手实验环节，带领读者一步步完成相关操作。
    * 理论先行，先讲清楚原理，再进行实验验证。
    * 对于实验中出现的关键且复杂的命令，务必详细解释其每个参数的含义和作用，避免读者知其然不知其所以然。
4.  **互动与迭代：**
    * 这是一个多轮对话任务。每轮对话产出一篇以某个技术点为核心的博客文章。
    * 我会对每篇文章进行阅读，并可能提出疑问或要求修改。你需要在我满意后继续下一篇的撰写。
5.  **风格与格式：**
    * 保持专业的技术博客风格，行文流畅，逻辑清晰，解释准确。
    * 避免写成刻板的课程教案或操作手册。
    * 最终，我可能会要求你将所有博客文章汇总成一个完整的 Markdown 文档。请确保 Markdown 格式的正确性，尤其注意代码块、列表、标题等的渲染，以便在 Typora 等编辑器中能良好显示。
6.  **特定重点（可选）：**
    * 在学习 **[在此处填写具体技术主题]** 的过程中，我对 **[在此处填写你特别想深入理解的技术点，例如：iptables与路由、调度算法的优先级与抢占机制、自定义Hooks的闭包陷阱等]** 部分特别感兴趣，希望你能在这方面进行更详尽和透彻的讲解。

# 任务启动

请以上述要求为准，开始撰写系列博客的第一篇文章。文章开头可以有一个简短的系列介绍。如果需要我先提供学习材料，请告知。

（在后续对话中，我会表达出“继续下一篇”的意思或提出具体问题。）
```
